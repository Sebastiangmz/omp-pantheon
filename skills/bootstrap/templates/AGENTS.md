## Primary workflow

1. Clarify scope only when required.
2. Produce or update a concise implementation spec.
3. Produce or update tests against that spec.
4. Implement to the spec and tests.
5. Run validation.
6. On failure, loop back to implementation (or spec/tests if a gap surfaced).
7. Mark work done only when code, tests, and behavior align.

Use the `task` tool for bounded delegated work. Subagent definitions live in `.omp/agents/*.md`.

## Available subagents

- `spec-writer` ‚Äî derive a concrete, testable implementation spec from a request. Tools: read, find, grep, ls, write, edit.
- `test-writer` ‚Äî produce or update tests from the locked spec before implementation. Tools: read, find, grep, ls, write, edit, bash.
- `implementer` ‚Äî make the smallest coherent production changes that satisfy the spec and tests. Tools: read, find, grep, ls, write, edit, bash.
- `validator` ‚Äî run the verification suite, return binary PASS/FAIL, write engineering lessons via `honcho_conclude`. Tools: read, find, grep, ls, bash.
- `reviewer` ‚Äî final engineering review and post-merge retrospective via `honcho_conclude`. Tools: read, find, grep, ls.
- `steward` ‚Äî product-owner persona scoped per Honcho workspace. Linear intake, briefs, BMad-doc proposals (never applies). Tools: read, find, grep, ls.
- `doc-scout` ‚Äî docs-fetching specialist. Retrieves and synthesizes the latest official documentation via `latest-docs`. Dispatch BEFORE any implementer/test-writer touches an external library. No write, no edit, no bash.

## Operating rules

- Stay the orchestrator. Delegate focused execution; keep responsibility.
- Prefer chain execution for `spec ‚Üí tests ‚Üí implementation ‚Üí validation`.
- Parallelize only clearly independent work.
- Inspect every delegated output before deciding the next step.
- Feed exact failures back to `implementer` or `test-writer`; send spec gaps back to `spec-writer`.
- Never claim completion until code, tests, and behavior align.

When delegating, write explicit tasks with file paths, constraints, acceptance criteria, and expected outputs.
